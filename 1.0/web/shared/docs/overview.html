Summary:
Myna is Java web application (.war file) that provides dynamic web content from both interpreted and compiled JavaScript, as well as complete access to Java libraries.
Goals:
To provide an open source rapid web development platform that:
  • Can be distributed as a simple ".war" file.
  • Can leverage Java's large number of libraries.
  • Can be easily administered online.
  • Provides a rich standard library to speed development.
  • Provides easy access to databases.
  • Can run on JVM 1.5+
  • Can be deployed on top of any Java Servlets container
  • Can be distributed freely. All distributed code coverd under Open Source Licenses.
... but does NOT:
  • Provide an MVC framework. Developers are free to implement their own framework if appropriate.
  • Require Java programing to extend. As much behavior as practical will be implemeneted in JavaScript.

Myna Specific Features:
JVM 1.4+ compatibility: The Rhino API is accessed directly rather than through Java 6's scripting API.
Minimal setup: Dynamic pages can be created immediately on deployment of the .war file. There is no need to to set up complicated application definitions.
Server-side JavaScript : Files in the in the application root or sub folders ending with ".sjs" are treated as server-side JavaScript. JavaScript content is executed server side and sent to there browser. 
$res.print("<html>");
$res.print("	<body>");
$res.print("	Today's date is" + String(new Date()) + "!");
$res.print("	</body>");
$res.print("</html>");
Embedded JavaScript : Files in the in the application root or sub folders ending with ".ejs" are treated as embedded JavaScript. Embedded JavaScript files can contain any text content (typically HTML) with JSP style escaped dynamic content. 
<% var today = new Date(); %>
<html>
	<body>
		Today's date is<%=today%>!"
	</body>
</html>
Embedded JavaScript Macros: Macros are html-like contructs that make loops and conditionals easier to implement in ".ejs" files. 
<@loop array="qry.data" element="curRow" index="rownum">
	<tr>
		<td class="">
			<%=rownum%>  
		</td>
		<td class="">
			<%=curRow.first_name%>  
		</td>
		<td class="">
			<%=curRow.last_name%>  
		</td>
		<td class="">
			<@if curData.type == "A">
				Administrator
			<@elseif curData.type == "M">
				Manager
			<@else>
				Non-Management
			</@if>
		</td>
	</tr>
</@loop>
Embedded JavaScript Blocks : EJS blocks work like .ejs paged but can be used in JavaScript code to return a string. All text between and will be returned as if it was included from a .ejs page. This makes it much easier to embed other languages like SQL or HTML in JavaScript strings 
var qry = new Myna.Query({
  dataSource:"ds_name",
  sql:<ejs>
    select

      channum = <%=$req.data.channel_number%> 
      <@if !includeDeleted>
      and deleted != 1

  </ejs>
})
E4X support : http://developer.mozilla.org/en/docs/E4X_tutorial
Easily import, manipulate and export XML data from javascript
Standard library : Structured library of useful functions. Client side compatible functions are stored in seperate files so they can be loaded in the browser. See libOO JS docs
Standard Objects : Pre-defined objects that contain useful information about the envirinment. See standard_object.sjs
Application lifecycle hooks See $application standard object
Easy Multi-line Queries See Myna.Query
Database connection pooling : Database connections are re-used to to reduce overhead. Future releases will allow the maximum size/age of connection threads to be managed in the administrator.
Object-relational Mapping (ORM) : Rows in Database tables can be treated like objects and extended. See Myna.DataManager
Profiling : A global profiler object is available for profiling code. An administrator setting can display an execution time summary at the bottom of every page.
Script Compilation : JavaScript files can be compiled and cached to improve performance
Web-based Administration : All currently implemented settings can be modified through the provided administrator application
Easily extensible: Most of the framework for request handling is implemented in JavaScript, so it is possible to completely re-architect the request lifecycle without re-compiling the Java servlet.
Centralized datasources : Datasources can be defined in the administrator and then referred to by name at runtime
Buffered content: Generated content can be retrieved and cleared at any time during page execution, and is sent to the browser at the end of the request in a manner similar to PHP. This allows cookies and other headers to be sent at any time during the request. See $res standard object
Normalized Paths: All file handling us done using String URIs. This allows easy cross-platform file access. Also file names that are not absolute are expanded relative to either the application root, or the currently running script. See Myna.File and MynaPath
LDAP Queries: See Myna.Ldap
Strong Encryption: See String.toHash and String.encrypt
