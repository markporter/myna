Upgrading Myna will usually require a context reload or JVM restart so you may want to limit connections by setting Max Request to 0 in General Settings.

The upgrade process for myna consists of of 4 steps:

1. Upload new .war file

You can download the latest copy of Myna from http://sourceforge.net/project/showfiles.php?group_id=202389
Browse to the downloaded .war file and click "Start Upgrade" to start the upgrade

2. Examine files

In this step the system will determine what files in the upload war are actually different from the installed files. These new files will be output to /WEB-INF/upgrade_files. At the same time, a new folder is created under /WEB-INF/upgrade_backups/backup_<date and time>. This folder will contain the original version of any files replaced by the upgrade process.

3. Run upgrade script (if needed)

Any changes not related to simple file updates will be handled by the upgrade script included in the war file (/WEB-INF/upgrade.sjs)

4. Install upgraded files

Finally the new files will be copied to the webroot. If this upgrade replaced the java components, a context reload/JVM restart will be needed. This may happen automatically on some application servers.

Recovering from a failed upgrade

The first step when it appears you have a failed upgrade is to restart your application server. If you are still having problems, you can revert to your original version by copying the files in /WEB-INF/upgrade_backups/backup_<date and time> over the files in your webroot.
